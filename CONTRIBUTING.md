# Contributing to byte-space

## Vibes

No vibe coding. All contributions must be written by you, handcrafted and pretty, no slop please. 

The only exception is tui/ which is vibe coded.

## Conventions

Keep code simple and readable. Follow existing conventions in the codebase.

## How?

Fork the repo, make your changes on a new branch, and open a PR against main. Keep PRs focused, one feature or fix at a time.

To build the project run go build ./...

For bugs, open an issue with steps to reproduce and expected vs actual behavior.
